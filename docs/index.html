FuteBot

Aplicacao local para acompanhar jogos da Copa do Mundo, sincronizar resultados, consultar historico, simular torneios e gerar previsoes com Poisson, ELO e calibracao incremental.

Visao geral Setup Dados Modelo Telas QA Seguranca

Visao geral

O FuteBot roda como uma aplicacao Streamlit e usa SQLite local para armazenar selecoes, partidas, resultados sincronizados e snapshots de previsoes pre-jogo. O app funciona sem internet usando fallback local, mas tambem pode consumir Football-Data.org e OpenFootball.

Tempo real

Football-Data.org

Camada opcional para jogos ao vivo e partidas futuras quando ha token configurado.

Resultados

OpenFootball

Atualiza partidas finalizadas quando o JSON publico esta disponivel.

Previsoes

Poisson + ELO

Calcula xG, placar provavel, probabilidades 1X2 e simulacoes de torneio.

Setup local

Crie um ambiente virtual, instale as dependencias e rode o Streamlit.

python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
streamlit run app.py

Para testes locais, instale tambem as dependencias de desenvolvimento: 

pip install -r requirements-dev.txt

Fontes de dados e persistencia

  • SQLite local: criado automaticamente em data/futebot.db.
  • Dados seed: fallback offline para manter a UX quando fontes externas falham.
  • OpenFootball: usado para jogos finalizados de Copas quando acessivel.
  • Football-Data.org: usado para tempo real e agenda futura com token opcional.
  • Origem dos dados: partidas carregam origem_dados como seed, api, openfootball ou manual.

Modelo estatistico

Base

Forca ofensiva e defensiva

Calculada a partir de partidas finalizadas, com suavizacao para reduzir distorcao de amostras pequenas.

ELO

Ajuste de favoritismo

O diferencial de ELO altera os gols esperados antes da distribuicao de Poisson.

Calibracao

Aprendizado incremental

Previsoes pre-jogo sao salvas, avaliadas apos o placar real e usadas com limites conservadores.

A calibracao so entra quando ha amostra minima de previsoes avaliadas. Jogos seed, simulados ou sem placar real nao entram no aprendizado.

Telas principais

  • Home: partidas de 2026, status da API, previsoes pre-jogo, live model e simulador de jogo.
  • Estatisticas: estatisticas por selecao, simulacao de torneio e odds do proximo jogo.
  • Previsoes: confronto manual entre duas selecoes.
  • Historico: partidas antigas por edicao/fase, maior placar e cards com bandeiras.
  • Acuracia: backtest temporal e comparativo com previsoes calibradas avaliadas.

QA e verificacoes

Comandos recomendados antes de publicar alteracoes:

$env:PYTHONIOENCODING="utf-8"
python -m compileall app.py src pages
pytest -q

O projeto tambem possui testes para seguranca de token, status especiais, exclusao de seed na acuracia, integridade do banco e calibracao incremental.

Configuracao e seguranca

  • O token da API deve vir de FOOTBALL_DATA_API_KEY ou .streamlit/secrets.toml.
  • .streamlit/secrets.toml nao deve ser versionado.
  • O banco SQLite local em data/*.db tambem fica fora do Git.
  • Sem token, o app entra em modo offline e usa dados locais/fallback.
